## RSS Checker / Save workflow fix (evening)

### Root cause of #3-#12 save confusion
- RSS Feed Checker (9AM/9PM cron) was using `rss-check.sh` — a different script that did NOT write to `rss-last-digest.json`
- So "save #N" after a checker delivery used the morning digest cache — wrong articles

### Fix applied
- RSS Feed Checker cron updated to use `rss-digest.py` (same as morning digest)
- Both crons now write to `data/rss-last-digest.json` — single source of truth
- "save #N" will now always match whatever digest was most recently delivered
